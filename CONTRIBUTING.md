# Contributing

感谢你愿意改进 Maco Power Monitor。

## 开始之前

- 本项目优先保证真实数据来源，不接受模拟电池、功耗或适配器数据。
- 对系统无法稳定公开获取的指标，请明确标注限制，不要用推测值填补界面。
- 任何会增加常驻资源消耗的实现，都需要说明功耗和唤醒成本。

## 本地开发

```bash
swift build
swift run
```

打包测试：

```bash
./scripts/package_app.sh
open dist/MacoPowerMonitor.app
```

## 提交建议

- Issue 请尽量附上 macOS 版本、芯片型号、是否接电、是否在充电。
- 涉及功耗、适配器、电池健康的改动，请说明真实数据来源。
- UI 改动请附截图。
- 提交前请至少执行一次 `swift build`。

## 代码风格

- 保持模块化，避免把系统采集逻辑写进视图层。
- 尽量优先使用 Apple 系统接口和明确可解释的系统命令输出。
- 变量、函数和类型命名要清晰，避免含糊缩写。
- 新增数据项时，要补充 tooltip 或说明文本，确保用户能看懂。

## 不建议的改动

- 引入 Electron、Node 后台或高占用常驻运行时
- 引入未说明来源的估算功耗值
- 把需要管理员权限的数据改成默认后台持续提权
